# Mission: 42 Common Core — Libft

## Why
Prepare for the 42 Common Core by building Libft independently and defensibly. The immediate goal is not only to pass Moulinette, but to write C library functions whose contracts, edge cases, memory behavior, and build process can be explained during peer evaluation.

## Success looks like
- Implement every mandatory Libft function from its contract without copying a finished solution.
- Predict and test boundary cases involving NUL bytes, zero sizes, overlap, allocation failure, and integer limits.
- Build a correct `libft.a` with the required Makefile rules and compilation flags.
- Explain implementation choices and debug failures with a compiler, debugger, and memory checker.
- Progress from mandatory functions to bonus linked-list functions with clear ownership rules.

## Constraints
- Current focus is Libft; later Common Core projects should not dilute it.
- Exercises may be drawn from the selected GitHub repositories, but claims and function contracts must be checked against primary or authoritative sources.
- Starting level: completed and passed the 42 Piscine; basic C syntax and problem-solving should not be retaught without evidence of a gap.

## Out of scope
- Copying or submitting repository implementations.
- Optimizing for bonus work before the mandatory library is correct and explainable.
- Detailed preparation for later Common Core projects.
